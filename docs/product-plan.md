# ShadowTech Product Plan

## MVP Feature Notes

### TTS Transcript Highlighting

Generated TTS practice material should support synchronized word highlighting while the user listens and shadows.

Requirements:

- When generating TTS audio, capture word boundary timing metadata from the TTS provider.
- Store the generated audio in Cloudflare R2.
- Store the transcript timing metadata alongside the speech record, either in D1 for queryable lesson data or as a JSON object in R2 when the payload is larger.
- The mobile app should track the current audio playback position and highlight the active word where `startMs <= currentTimeMs <= endMs`.
- Offline downloads must include both the audio file and its timing metadata.
- Do not rely on forced alignment for MVP when the audio is generated by our own TTS flow. Forced alignment should only be a fallback for imported audio without timing metadata.

Example timing payload:

```json
{
  "text": "Hi, I'm Daniel. I just joined the product team.",
  "audioUrl": "r2://speech/audio.mp3",
  "words": [
    { "word": "Hi", "startMs": 0, "endMs": 220 },
    { "word": "I'm", "startMs": 230, "endMs": 420 },
    { "word": "Daniel", "startMs": 430, "endMs": 850 }
  ]
}
```

MVP acceptance criteria:

- A generated speech can be played with word-by-word highlighting.
- Highlighting stays in sync during play, pause, seek, and replay.
- Cached/offline speeches preserve the same highlighting behavior.
- Capturing word timing does not require a separate AI evaluation call.
